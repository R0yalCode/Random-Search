01 · Concepto Fundamental
¿Qué es Random Search?
En Machine Learning, un modelo tiene hiperparámetros — configuraciones que tú decides antes de entrenar: cuántos árboles, qué tan profundos, qué tasa de aprendizaje. El problema: hay miles de combinaciones posibles.

Random Search es una estrategia que selecciona un subconjunto de esas combinaciones de forma aleatoria dentro del espacio definido, las evalúa, y elige la mejor. No busca todas las opciones — las muestrea.
Configuración = (p₁, p₂, …, pₘ) Vector de hiperparámetros sampled aleatoriamente
k muestras → mejor resultado Solo k de las infinitas combinaciones posibles
💡
Intuición clave: Según Bergstra & Bengio (2012), en espacios de alta dimensión Random Search es más eficiente que Grid Search porque los hiperparámetros no contribuyen por igual al rendimiento — algunos apenas importan.
02 · Complejidad Computacional
O(n × C) — ¿Qué significa cada variable?
La complejidad de Random Search se expresa de forma simple y directa:
O(n × C) Complejidad total de Random Search
n
Número de muestras aleatorias. Tú eliges cuántas combinaciones probar. Es lineal: doblas n → doblas el tiempo.
C
Costo de evaluar UNA configuración. Incluye entrenar el modelo + calcular la métrica (accuracy, F1, etc.). Depende del hardware y datos.
n×C
Costo total real. Si n=50 muestras y C=10 segundos/modelo → 500 segundos totales. Completamente predecible.
Comparado con Grid Search → O(gᵈ × C):
g
Resolución de la grilla. Cuántos valores por hiperparámetro (ej: 5 valores de learning_rate).
d
Dimensiones / número de hiperparámetros. Con 3 hiperparámetros y g=5 → 5³ = 125 combinaciones.
gᵈ
Explosión combinatoria. d=5, g=5 → 3,125 evaluaciones. d=10, g=5 → ¡9.7 millones!
O(n · C)
Random Search
↑ LINEAL — Controlable
O(gᵈ · C)
Grid Search
↑ EXPONENCIAL — Peligroso
🎯
Conclusión: Random Search escala linealmente — tú controlas el presupuesto computacional con un solo número (n). Grid Search escala exponencialmente — se vuelve intratable con más de 4-5 hiperparámetros.
